Evaluate one document's PICO extraction quality with a structured rubric.

Task constraints:
- Use only the provided abstract, gold texts, and predicted texts.
- Do not use external knowledge.
- Judge semantic alignment, not surface-form overlap.
- Prefer meaning, study-role correctness, and scope over exact wording.
- Do not reward token overlap if the meaning is wrong.
- Do not penalize minor wording differences if the predicted text is semantically equivalent to the gold evidence in the abstract.
- Use EBM-NLP framing where I means intervention plus comparator/control mentions when present.
- Audit P, I, and O separately before writing the overall judgment.
- Output JSON only.

Document id: {doc_id}

Abstract:
{abstract}

Gold texts:
- P: {gold_p}
- I: {gold_i}
- O: {gold_o}

Predicted texts:
- P: {pred_p}
- I: {pred_i}
- O: {pred_o}

Label audit rules:
- P = participant/population/study-group mentions.
- I = intervention, treatment arm, and comparator/control mentions when present.
- O = outcomes, endpoints, measured effects, or assessment targets.
- Judge each label independently; strong performance on one label must not improve another label's score.
- If a gold label is empty, an empty prediction is correct; non-empty predictions count as noise and inaccuracy.

Rubric values and definitions:
- completeness: `complete` = captures all major gold semantics for that label; `partial` = captures only some gold semantics; `missing` = captures none of the gold semantics or leaves the label effectively absent.
- accuracy: `accurate` = predicted text is semantically correct for the label; `mixed` = partly correct but includes semantic drift, ambiguity, or only some correct items; `incorrect` = semantically wrong for the label.
- noise: `low` = little irrelevant text; `medium` = some irrelevant, redundant, or loosely related text; `high` = substantial irrelevant or misleading text.
- granularity: `appropriate` = span scope matches the semantic unit; `too_broad` = includes extra context beyond the target unit; `too_narrow` = misses key parts of the target unit; `mixed` = multiple predictions show inconsistent scope.
- overall_verdict: `strong` = good semantic match with acceptable completeness and low error burden; `mixed` = usable but noticeably incomplete or inconsistent; `weak` = poor semantic match, major omissions, or substantial noise/errors.

Output format:
- Return one JSON object with top-level keys `labels` and `overall`.
- `labels` must contain `P`, `I`, and `O`.
- Each of `labels.P`, `labels.I`, `labels.O`, and `overall` must contain: `completeness`, `accuracy`, `noise`, `granularity`, `overall_verdict`, `reason`.
- Each `reason` must be exactly one concise sentence based on semantic evidence from the abstract, gold texts, and predicted texts.
