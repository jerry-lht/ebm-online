Task definition:
Extract outcome spans from the randomized controlled trial abstract below.

O definition and scope:
- O means outcome mentions only: endpoint names, measurement targets, scales, laboratory or biomarker measurements, symptom names, adverse-event names, or other noun phrases that directly name what was measured.
- If multiple distinct outcome mentions appear in the abstract, extract each exact mention separately.

Rules:
- Extract only exact substrings from the abstract.
- Do not paraphrase or infer.
- Return only JSON in this exact format: `{{"outcomes":["..."]}}`
- Example with one span: `{{"outcomes":["pain score"]}}`
- Example with multiple spans: `{{"outcomes":["pain score","hospital stay"]}}`
- If no explicit O span is stated, return `{{"outcomes":[]}}`.

Hard rules for O:
- Extract the shortest exact noun phrase that can stand alone as the outcome name.
- If the abstract contains both an outcome name and a result statement, keep only the outcome name.
- Extract `pain score`, not `pain score was lower`.
- Extract `plasma cortisol levels`, not `plasma cortisol levels increased after surgery`.
- Extract `nasal-breathing-resistance according to scores`, not `clinically proven efficacy`.
- Extract `natural killer (NK) cell activity`, not `recovered on postoperative day 2`.
- If a sentence only expresses efficacy, superiority, significance, improvement direction, or an author conclusion without a standalone outcome mention, do not output O for that text.
- Do not output O items such as `effective`, `superior`, `beneficial`, `more beneficial`, `significant change`, `no significant changes`, `improved`, `reduced`, `increased`, `recovered`, or full result statements like `X was lower`, `X was higher`, or `X improved`.
- Do not output mechanism or efficacy summaries from conclusion sentences unless they contain a standalone outcome mention.
