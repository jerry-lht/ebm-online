Task:
- You are performing full-text screening for one candidate study against one systematic review
- First read the review question and the raw review criteria prose
- Then read the candidate study title, abstract, and full text
- Derive the review's inclusion and exclusion criteria from the raw criteria prose
- Output criterion-wise judgments only

Requirements:
- Return exactly one JSON object and nothing else
- No markdown fences
- No prose before or after the JSON
- The only top-level key is `criterion_judgments`
- Under `criterion_judgments`, create keys `inc_1`, `inc_2`, ... for inclusion criteria
- Under `criterion_judgments`, create keys `exc_1`, `exc_2`, ... for exclusion criteria
- Every criterion value must contain exactly two keys: `text`, `judgment`
- `text` must restate the criterion briefly
- `judgment` must be one of `yes`, `no`, `unclear`
- Do not output a top-level include or exclude decision
- Do not output `reason`
- Do not output `evidence_spans`
- Do not output `criterion_group`
- If information is missing or ambiguous, use `unclear`
- Inclusion semantics:
  - `yes` means the study satisfies the inclusion criterion
  - `no` means the study does not satisfy the inclusion criterion
- Exclusion semantics:
  - `yes` means the study matches the exclusion criterion
  - `no` means the study does not match the exclusion criterion

Input:
Review question: {question}

Example ID:
{example_id}

Input setting:
{input_setting}

Raw review criteria:
{raw_review_criteria}

{candidate_block}

JSON template:
{json_template}
