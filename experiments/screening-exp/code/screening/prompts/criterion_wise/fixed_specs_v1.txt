Task:
- You are performing full-text screening for one candidate study against one systematic review.
- Judge only the fixed criteria listed below.
- Do not create, remove, rename, merge, or split criteria.
- Return one judgment for every listed `inc_*` and `exc_*` criterion ID.
- Do not output criteria IDs that are not listed.

Requirements:
- Return exactly one JSON object and nothing else.
- No markdown fences.
- No prose before or after the JSON.
- The only top-level key is `criterion_judgments`.
- Every criterion value must contain exactly two keys: `text`, `judgment`.
- `text` must briefly restate the listed criterion.
- `judgment` must be one of `yes`, `no`, `unclear`.
- Do not output a top-level include or exclude decision.
- If information is missing or ambiguous, use `unclear`.
- Inclusion semantics:
  - `yes` means the study satisfies the inclusion criterion.
  - `no` means the study does not satisfy the inclusion criterion.
- Exclusion semantics:
  - `yes` means the study matches the exclusion criterion.
  - `no` means the study does not match the exclusion criterion.

Input:
Review question: {question}

Example ID:
{example_id}

Input setting:
{input_setting}

Fixed review criteria:
{criterion_block}

{candidate_block}

JSON template:
{json_template}
