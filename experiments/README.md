# Experiments

这个目录专门放临时或半正式实验，避免把实验脚本、实验数据和实验结果散落到仓库根目录或 `docs/`。

建议约定：

- `code/`：实验脚本、notebook、一次性分析代码
- `data/`：实验输入数据、小型样本、实验配置
- `results/`：实验输出、日志、图表、导出 JSON/CSV

命名建议：

- 每个实验单独建子目录，例如 `code/2026-05-13-phase5-trace-check/`
- 数据和结果目录尽量与实验名对应，便于清理和复现

注意：

- 大文件不要直接提交；必要时在子目录继续补 `.gitignore`
- 进入正式产品代码或正式文档后，应从实验目录迁移到 `backend/`、`frontend/`、`docs/` 等正式位置
