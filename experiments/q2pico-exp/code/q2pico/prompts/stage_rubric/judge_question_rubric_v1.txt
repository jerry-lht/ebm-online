Evaluate one clinical question's PICO slot extraction quality with a structured rubric.

Task constraints:
- Use only the provided clinical question, gold slots, and predicted slots.
- Do not use external knowledge.
- Judge semantic adequacy, not only surface overlap.
- Prefer meaning, role correctness, and scope over exact wording.
- Output JSON only.

Question id: {question_id}

Clinical question:
{question_text}

Gold slots:
{gold_slots}

Predicted slots:
{predicted_slots}

Label audit rules:
{label_audit_rules}
- Judge each label independently; strong performance on one label must not improve another label's score.
- Judge only these active labels: {label_list}.
- If a gold label is empty, an empty prediction is correct; non-empty predictions count as noise and inaccuracy.

Rubric values and definitions:
- completeness: `complete` = captures all major gold semantics for that label; `partial` = captures only some gold semantics; `missing` = captures none of the gold semantics or leaves the label effectively absent.
- accuracy: `accurate` = predicted slot is semantically correct for the label; `mixed` = partly correct but includes semantic drift, ambiguity, or only some correct items; `incorrect` = semantically wrong for the label.
- noise: `low` = little irrelevant text; `medium` = some irrelevant, redundant, or loosely related text; `high` = substantial irrelevant or misleading text.
- granularity: `appropriate` = slot scope matches the semantic unit; `too_broad` = includes extra context beyond the target unit; `too_narrow` = misses key parts of the target unit; `mixed` = multiple predictions show inconsistent scope.
- overall_verdict: `strong` = good semantic match with acceptable completeness and low error burden; `mixed` = usable but noticeably incomplete or inconsistent; `weak` = poor semantic match, major omissions, or substantial noise/errors.

Output format:
- Return one JSON object with top-level keys `labels` and `overall`.
- `labels` must contain exactly these active labels: {label_list}.
- Each active label object and `overall` must contain: `completeness`, `accuracy`, `noise`, `granularity`, `overall_verdict`, `reason`.
- Each `reason` must be exactly one concise sentence based on the clinical question, gold slots, and predicted slots.
